Raw APIs vs platforms when each one wins
Platforms (Vapi, Retell, Bland) resell the same models you can call directly: you pay ~$0.05–0.10/min for a config UI, telephony, and not writing code. Worth it below ~1,000 min/month or when the client owns nothing. Raw APIs win when you need custom tools against real systems (databases, calendars, trading engines), sub-vendor costs at scale, or you're selling engineering credibility. The desk on this site is raw: speech-to-speech over WebRTC, tools executing server-side, one ephemeral key per session. Rule of thumb: prototype on either, but if the demo IS the product, go raw and own it.
Personas that sound human what we un-learned the hard way
Human ≠ hesitant. Our v1 mandated reaction beats ("oh, yeah…") and engineered mistakes; it read as a broken machine within a day of real calls. What works: confidence as the first rule (direct first words, zero warm-up sounds), filler words as an explicit dial set to "occasionally" or "none", a variety rule (never the same sentence twice), and OpenAI's persona skeleton: Role & Objective, Personality dials, Context, Reference Pronunciations, numbered Conversation Flow with sample phrases, Safety. Every dial is in agents/06_voice/personas.py in the repo.
Turn-taking & noise the #1 demo killer
Coughs, keyboards and mumbles will trigger your agent mid-silence and users hate it more than any other flaw. Three-layer fix: semantic VAD with eagerness low (waits for end-of-thought, not end-of-sound), an explicit prompt rule (noise is not speech; the human response to noise is silence, at most one casual "you say something?"), and a client-side idle hang-up (3 min) so an abandoned mic doesn't babble or bill.
Sound design foley yes, drone no
Dead digital silence between words screams bot, but a synthetic "office ambience" loop reads as traffic noise. What survived user testing: keyboard foley tied to real tool calls (she says "one sec, pulling that up" and you hear actual typing while the API call runs) plus occasional idle keystrokes. Procedurally generated in ~60 lines of Web Audio, no audio files. Skip the room tone.
Security for public agents the part kits never mention
A public voice agent is an open prompt to your infrastructure. Non-negotiables: ephemeral per-session keys minted server-side (the real key never reaches the browser), all tools execute server-side, read-only DB access with keyword guardrails, spend caps on every API key, token-gated webhooks, hourly budget guards on expensive endpoints, and never, ever a personal login wired into a public agent: anything the agent can see, any caller can extract by asking.
Prove it works traces, tests, evals
A demo that worked once is not a product. The desk ships with LangSmith traces on every run (including voice tool calls), 48 CI-gated tests (pricing math, SQL injection guardrails, graph topology), and graded evals: text-to-SQL is scored by executing reference queries against a deterministic DB, no LLM judge needed. When a client asks "how do you know it still works?", you show the dashboard, not a shrug.
