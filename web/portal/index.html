The estate, as one system

Seven services, ten boxes, 44 processes and stores. Every line is something that actually runs, and the ones that are wrong are drawn wrong on purpose: a service reaching into another service's database is red, and a link that was designed and never wired is a ghost. Click a box to open it up.

level 0 · one box per service

Why nothing moves. A map like this usually animates a request travelling across it. This one does not, because it would be a lie: there is no trace id anywhere in the estate that survives a service boundary. The bank's x-ray works only because one ledger transaction id happens to be reused verbatim on every shard, which is a property of that service, not a mechanism. Until every Kafka payload carries an envelope with a trace id and every cross-service call forwards a header, a journey cannot be followed across these boxes, so none is drawn.
